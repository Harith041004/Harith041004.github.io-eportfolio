Major Projects

Project 1: VitalityApp Prototype

Problem: Many individuals struggle with inconsistent health management because they lack a single, centralized tool to track physical habits and mental reflections.

Lessons Learned: Successfully built a multi-screen application using Jetpack Compose and Navigation. I learned how to share and hoist state using a centralized ViewModel to ensure user data survives screen rotations.

Project 2: VitalityApp Ecosystem

Problem: Expanding on Project 1, users need their fitness actions verified and synced remotely to stay motivated within a community.

Lessons Learned: Integrated advanced hardware sensors using Google ML Kit (CameraX Face Detection) to track push-ups. I successfully connected a local Room database with remote Firebase Firestore and utilized Retrofit to fetch live REST API data.

Reflection & Learning Journey

From Lab 1 to Project 2, my journey evolved from building simple static UIs to developing a complete, data-driven Android ecosystem. I overcame significant technical challenges, particularly in managing complex state across 7 screens and implementing advanced hardware sensors via Google ML Kit.

Building VitalityApp demonstrated how mobile technology can directly impact SDG 3 by fostering better health habits in the community. Learning to integrate Room for local persistence, Firebase for cloud syncing, and Retrofit for live APIs has given me the confidence and skills to architect production-ready software.
